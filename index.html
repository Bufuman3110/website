Comandi CMD e Strumenti di Sistema

CMD

  • netsh wlan show profiles - Visualizza le reti memorizzate.
    netsh wlan show profiles “ nome rete” key=clear - Visualizza le credenziali reti memorizzate; su contenuto chiave trovi la password della rete.
  • driverquery - serve a fare la lista dei driver del computer con tutti i dettagli.
    Scrivere driverquery-v
    driverquery-v > listadriver-txt salverà la lista in un file di testo nella cartella corrente.
  • shutdown spegne il computer.
    shutdown /r /o riavvia il PC e lancia il menu di opzioni avanzate di avvio per accedere alla modalità provvisoria e all' utilità di recupero.
  • sfcsta per System File Checker e serve a fare il controllo dei file di sistema per correggere errori di file non trovati o corrotti.
    sfc /scannow rileva file danneggiati o mancanti e li sostituisce con file buoni.
  • tasklist - Mostra le attività in esecuzione.
    tasklist-svc mostra i servizi relativi a ciascuna attività,
    tasklist-v fornisce maggiori dettagli su ogni processo e
    tasklist-m individua i file dll con processi attivi. Questi comandi sono utili per la risoluzione avanzata dei problemi o per rintracciare manualmente malware.
  • taskkill /im nome.exe - Termina un processo.
  • assoc per vedere le associazioni dei file con i programmi.
    assoc .txt per vedere quale programma apre i file txt.
    Scrivendo assoc .txt= si potrà cambiare il programma da usare.
  • fc sta per File Compare e serve a confrontare due file di testo per vedere se ci sono piccole modifiche tra due versioni di uno stesso file.
    Digitare quindi fc e poi percorso della cartella e nome dei due file che si vogliono confrontare.
    Aggiungendo l' opzione /l per confrontare solo il testo ASCII.
  • ipconfig - Visualizza l'indirizzo IP del computer.
    ipconfig /renew è il comando per chiedere un nuovo indirizzo IP, utile per ripristinare internet in caso di problemi con l'ip.
    È anche possibile utilizzare ipconfig /flushdns per aggiornare la risoluzione dei nomi DNS.
    ipconfig /all mostra tutti gli indirizzi di rete del computer compreso il mac.
    Per linux è ifconfig
  • netstatInserendo il comando netstat-an si otterrà la lista delle porte di rete attualmente aperte con i relativi indirizzi IP.
    Questo è un grande comando per cercare e risolvere problemi di malware o collegamenti a internet dei software.
  • ping www.google.it - Controlla la connessione a Internet con dispositivi interni ed esterni alla lan
  • pathping è una versione più avanzata di ping utile se ci sono più router tra il PC e il dispositivo che si sta testando.
  • tracert indirizzo per sapere tutti i passaggi che fa un pacchetto per arrivare a quell' indirizzo partendo dal nostro computer.
    A differenza pathping, tuttavia, tracert mostra anche quanto tempo passa (in millisecondi) ogni passaggio tra server o dispositivi.
  • powercfg è un comando molto potente per la gestione dell' energia.
    Si può utilizzare il comando powercfg /hibernate on e powercfg /hibernate off per gestire la modalità di ibernazione ed è anche possibile utilizzare il comando powercfg /a per visualizzare gli stati di risparmio energetico disponibili sul PC.
    Un altro comando utile è powercfg /devicequery s1_supported che vedere la lista di dispositivi che possono risvegliare il computer dallo stato di sospensione.
    powercfg /lastwake invece ci dice qual è l' ultimo dispositivo che ha svegliato il PC da uno stato di sospensione.
    Il comando powercfg /energy può essere usato per avere una dettagliata relazione sul consumo di energia del PC.
    Questo rapporto è utile a verificare eventuali anomalie di sistema che possono aumentare il consumo di energia, in Windows 8 si può usare anche il comando powercfg /batteryreport per avere un' analisi dettagliata dell' uso della batteria nei portatili.
  • recimgsta per Recovery Image e si può eseguire solo in Windows 8 per creare un' immagine di sistema. Praticamente tutti i computer Windows comprati già installati hanno già un' immagine di ripristino per fare il reset ma questa potrebbe includere programmi inutili che si preferirebbe non avere. Eseguendo recimg si ottiene una lista di opzioni per usare questo comando in modo da creare un' immagine personalizzata del sistema.
  • net user Mostra gli utenti registrati (anche in passato) nel pc con le rispettive password
    sintassi del comando NET USER: 
            [nomeutente [password | *] [opzioni]] [/DOMAIN]
        nomeutente { password | * } /ADD [opzioni] [/DOMAIN]
        nomeutente [/DELETE] [/DOMAIN]
        nomeutente [/TIMES: { times | ALL } ] 
        nomeutente [/ACTIVE: { YES | NO } ]
  • dcomcnfg consente di configurare varie impostazioni specifiche di DCOM nel Registro di sistema. La finestra DCOMCNFG include tre pagine:
    • Sicurezza predefinita
      La pagina Sicurezza è simile alla pagina Sicurezza predefinita disponibile nella finestra DCOMCNFG, con la differenza che queste impostazioni si applicano solo all' applicazione corrente. Anche in questo caso, le impostazioni vengono archiviate nell' AppID per l' oggetto. La pagina Identifica identifica l' utente usato per eseguire l' applicazione. La pagina Endpoint elenca il set di protocolli ed endpoint disponibili per l' uso da parte dei client del server DCOM selezionato.
    • Proprietà predefinite
      È necessario selezionare la casella di controllo Abilita Distributed COM in questo computer se si desidera che i client di altri computer accedano agli oggetti COM in esecuzione nel computer. Se si seleziona questa opzione, HKEY_LOCAL_MACHINE\Software\Microsoft\OLE\EnableDCOM il valore viene impostato su Y.
    • Applicazioni
      Modifica le impostazioni per un determinato oggetto, è sufficiente selezionare l' applicazione dall' elenco e fare clic sul pulsante Proprietà . La Finestra Proprietà ha cinque pagine:
      • La pagina Generale conferma l' applicazione in uso.
      • La pagina Percorso consente di specificare dove deve essere eseguita l' applicazione quando un client chiama “ CoCreateInstance ” il CLSID pertinente. Se si seleziona la casella di controllo Esegui applicazione nel computer seguente e si immette un nome di computer, “ RemoteServerName” viene aggiunto un valore in AppID per l' applicazione. Se si deseleziona la casella di controllo Esegui applicazione nel computer “ LocalService_LocalService” , il valore viene rinominato e quindi disabilitato.
    • Protocolli predefiniti (solo Windows 2000)
      Elenca il set di protocolli di rete disponibili per DCOM in questo computer. L' ordine riflette la priorità in cui verranno usati; il primo nell' elenco ha la priorità più alta. I protocolli possono essere aggiunti o eliminati da questa pagina.
  • mkdir /md crea una cartella
  • move sposta una cartella.
  • rd /s elimina una cartella
  • > è utilizzato per reindirizzare il messaggio dallo schermo a un file, generalmente è utilizzato nel comando echo
  • more o type sono utilizzati per visualizzare il contenuto dei file di testo appena creati.
